# BijecBet Website

BijecBet is a React + Vite marketing site for a data-driven arbitrage platform.  
It includes animated hero visuals, sportsbook monitoring UI, comparison sections, a profit calculator, FAQ, and a Supabase-backed waitlist form.

## Tech stack

- React 19
- Vite 8
- Tailwind CSS 3
- Framer Motion
- React Hook Form + Zod
- Supabase (`@supabase/supabase-js`)
- Wrangler / Cloudflare deployment tooling

## Sections in the app

- Sticky Navbar
- Background Banner
- Hero (with live market feed)
- How It Works
- Features
- Manual vs. Bijec comparison
- Profit Calculator
- Trust & Risk
- Monitored Sports Books
- FAQ
- Final CTA + waitlist form
- Footer

## Environment variables

Create a local `.env` (or copy from `.env.example`) with:

```bash
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
VITE_GA_MEASUREMENT_ID=G-XXXXXXXXXX
```

`VITE_GA_MEASUREMENT_ID` is optional (analytics only).

## Supabase waitlist setup

The waitlist form inserts into `public.waitlist` with fields: `email`, `source`.

Run this in Supabase SQL Editor:

```sql
create table if not exists public.waitlist (
  id bigint generated by default as identity primary key,
  email text not null,
  source text not null default 'hero',
  created_at timestamptz not null default now()
);

create unique index if not exists waitlist_email_unique_idx
  on public.waitlist (lower(email));

create index if not exists waitlist_created_at_idx
  on public.waitlist (created_at desc);

create index if not exists waitlist_source_idx
  on public.waitlist (source);

alter table public.waitlist enable row level security;

drop policy if exists "anon_insert_waitlist" on public.waitlist;
create policy "anon_insert_waitlist"
on public.waitlist
for insert
to anon
with check (true);

drop policy if exists "anon_select_waitlist_denied" on public.waitlist;
create policy "anon_select_waitlist_denied"
on public.waitlist
for select
to anon
using (false);
```

## Deployment notes

- Build command: `npm run build`
- Deploy command: `npx wrangler deploy` (or `npm run deploy`)
- If CI fails on `npm ci` with lockfile mismatch, regenerate and commit `package-lock.json`.
- If cloud builds show stale dependency errors, clear build cache and redeploy.
